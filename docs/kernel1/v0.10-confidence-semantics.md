# Kernel1 v0.10 — Confidence 语义合约

**日期:** 2026-05-16
**范围:** 重新定义 `result.confidence` 的含义,把 arbitration 降级为建议层,在 element_dimensions 上强制 Model A 约束。

---

## 1. 一句话

`result.confidence` 不再是 Top-1 raw score,而是系统对最终输出 `type` 的可信度。
**只有当 status=="certain" 且算法 Top-1 与综合层一致时,confidence 才能 ≥ 0.80。**

---

## 2. v0.9 → v0.10 字段映射

### 2.1 confidence 含义变了

| 旧(v0.9) | 新(v0.10) |
|---|---|
| `confidence = top["score"]` | `confidence = _calibrate_confidence(top, status, saturation_event, partial_recovered, arbitration_decision)` |
| 撞顶 case (Top-3 全 1.0) → 显示 1.0 | 撞顶 → 强降到 `saturation_confidence` (默认 0.4) |
| status=clarifying + confidence=1.0 可共存 | status!=certain → `min(raw*0.6, uncertain_confidence_cap)` (默认 0.5) |
| arbitration disagree → 重排 candidates + 改 type | arbitration disagree → status 强制 uncertain,confidence ≤ 0.4 |
| partial JSON recover → cap 0.74 | partial recover → 同样 cap 0.74(保留) |

### 2.2 新字段

| 字段 | 含义 |
|---|---|
| `confidence_breakdown.raw_score` | 算法 Top-1 的归一化分数(没校准前) |
| `confidence_breakdown.reason` | 校准触发的所有规则,`+` 串联(如 `status_uncertain+saturation_top2_collision`) |
| `confidence_breakdown.saturation_event` | 撞顶事件详情(若有):`{event, top_scores, spread_top1_top2, band}` |
| `algorithm_top.type` / `algorithm_top.score` | 算法层 Top-1 不变事实;arbitration 不能改这两个字段 |
| `arbitration.suggested_type` | 综合层 verdict;只展示,不改 result.type |
| `arbitration.agrees_with_algorithm` | bool,综合层与算法 Top-1 是否一致 |
| `arbitration.decision` | `agree` / `disagree_supported` / `disagree_unsupported` |
| `stability_hint.algorithm_top` | 算法 Top-1(冗余字段,方便前端) |
| `stability_hint.synthesis_verdict` | 综合层 verdict |
| `stability_hint.coherence` | 综合层置信度(0-1) |
| `stability_hint.consistent` | bool,综合与算法是否一致(consistent=agrees && coherence>=0.75) |

### 2.3 校准规则(`_calibrate_confidence`)

按以下顺序累加(多条件取最严格 = 最低值):

```
raw = top.score

if status != "certain":
    raw → min(raw * 0.6, uncertain_confidence_cap)   # 默认 cap=0.5

if saturation_event (撞顶):
    raw → min(raw, saturation_confidence)            # 默认 0.4

if arbitration_decision == "uncertain"(等价 disagree):
    raw → min(raw, saturation_confidence)            # 默认 0.4

if partial_recovered:
    raw → min(raw, partial_recovered_cap)            # 默认 0.74
```

### 2.4 撞顶检测(`_detect_saturation`)

按 ranked 列表前 3 个候选的 score:

| 情境 | event |
|---|---|
| Top-1, Top-2 都 ∈ [0.95, 1.0] 且 Top-1 - Top-2 < 0.05 | `top2_collision` |
| 上面满足 + Top-3 也 ∈ [0.95, 1.0] 且 Top-1 - Top-3 < 0.05 | `top_band_collision` |
| 其他 | None |

**单点 1.0**(Top-2 < 0.95)不触发,保护合法的"高分独占 Top-1"场景。

### 2.5 多 4D 截断(`_enforce_dimension_constraints`)

`_synthesize_evidence` 拿到 LLM 的 `element_dimensions` 决策表后,扫描:

- **4D 元素** > `max_4d_per_person`(默认 2):按 `vote_4d` 降序保留 Top-K,余下强制降为 3D。Model A 一人最多 2 个 4D(主导 1 号位 + 演示 8 号位)。
- **1D 元素** > 1:按 `vote_1d` 降序保留最强,余下升为 2D。Model A 一人最多 1 个 1D(薄弱 4 号位)。

每条降维记录追加到 `synthesis_notes` 的 `[自动降维:...]` 段。

## 3. AnalyzeOptions 新增字段

```python
saturation_top_band: tuple[float, float] = (0.95, 1.0)
saturation_max_spread: float = 0.05
uncertain_confidence_cap: float = 0.5
saturation_confidence: float = 0.4
partial_recovered_cap: float = 0.74
max_4d_per_person: int = 2
```

`target_confidence = 0.80` 仍存在,但语义现在是"系统给出 type 的可信度门槛",不是"score 高门槛"。

## 4. 前端徽标语义(GUI)

`static/index.html` 上 `mConfBadge` 颜色规则:

| 条件 | 颜色 | 文案 |
|---|---|---|
| `status=="certain"` && `confidence >= 0.80` | 绿 #0a0 | `0.xxx ✓` |
| `status=="certain"` && `confidence < 0.80` | 黄 #c80 | `0.xxx ✓` |
| 其他(status != certain) | 红 #c00 | `0.xxx (无 type)` |

`title` 属性显示 `raw_score` 和 `reason`(鼠标悬停可看)。

**关键变化**:绿色不再代表"分数高",代表"系统给出唯一 type 且达标"。撞顶 case 即使 raw_score=1.0,徽标也显示红色 + (无 type)。

新增徽标:
- `mAlgoTop` — 显示 `algorithm_top.type (score)`
- `mStability` — 显示一致性,绿/红/灰(未仲裁)

## 5. 仲裁层语义(v0.10)

```
_arbitrate(result, candidates, synthesis):
    if synthesis is None:
        result.arbitration = None
        return result

    verdict = synthesis.verdict_type
    coherence = synthesis.coherence_confidence
    algo_top = candidates[0].type
    agrees = (verdict == algo_top)

    if agrees:
        decision = "agree"
    elif coherence >= 0.75 and verdict in top-3:
        decision = "disagree_supported"
    else:
        decision = "disagree_unsupported"

    result.arbitration = {
        "suggested_type": verdict,
        "decision": decision,
        "agrees_with_algorithm": agrees,
        "voters": {...},
    }
    result.algorithm_top = {"type": algo_top, "score": ...}
    result.stability_hint = {
        "algorithm_top": algo_top,
        "synthesis_verdict": verdict,
        "coherence": coherence,
        "consistent": agrees and coherence >= 0.75,
    }

    # 关键:不一致或 coherence 低 → 强制 uncertain
    if not (agrees and coherence >= 0.75):
        if result.status == "certain":
            result.status = "uncertain"
            result.type / alias / quadra / model_a = None / []

    # 重算 confidence
    result.confidence = _calibrate_confidence(...)
```

`_arbitrate` 再也不修改 `result.candidates` 顺序、不改 `result.type`(除了强制清空)、不重排 explanations。算法层 Top-1 始终是真相,综合层只能背书或反对。

## 6. 回归保证

`kernel1/tests/test_confidence_calibration.py` 锁住下列断言:

- **`web-1778908013096.json`**(Top-3 全 1.0):confidence ≤ 0.5,saturation_event.event = `top_band_collision`,status = uncertain,type = None
- **`web-1778907758325.json`**(Top-2 撞顶):confidence < 0.8 当 status != certain,saturation reason 出现在 breakdown
- 仿 disagree 场景:arbitration 不覆盖 candidates 顺序,suggested_type ≠ type 同时存在
- 仿 agree 场景:stability_hint.consistent = True
- coherence < 0.75:即使一致也强制 uncertain

`kernel1/eval/run_synthetic_repeated.py`:同输入连跑 N 次,断言 verdict 一致率 ≥ ⌈N*2/3⌉ 或全 uncertain。跳变 case 退出码 1。
